## 新能源汽车

背景：车企现在想开发新型号的汽车，开发前要去查看市场对正在销售的汽车的评价

数据的来源：爬虫。懂车帝、汽车之家、小红书等途径，获得消费者对车的评价数据

功能开发：

* 用户更加看重什么方面：续航、外观、内部空间、马力等
* 细分：续航，短续航、中等续航、长续航
* 细分：比亚迪汽车的外观，用户的整体评价是好评、中评、差评
* 效果：能够指导下个车型各个方面如何升级优化，设计



## 项目名称：电商评论分析

项目描述及实现功能：基于服饰商品评论数据,训练模型对评论进行分类处理,对商家销售策略进行建议。

项目职责：

1.进行文本的数据清洗,使用bert模型进行训练和调优,准确率达到93%。

2.为了满足部署需求,使用textcnn模型进行知识蒸馏,准确率达到91%左右,模型占用空间大大减少,模型预测样本时间有很大提升.

3.对服饰商品的评论使用模型进行预测后,给予商家一些销售建议,包含对商品的上架及库存进行调整等,商品的销售额提升12%左右.

开发环境:Linux, python

## 项目名称：教学效果评价系统

项目背景：为了评估教师教学效果以及学员满意度，基于公司学生对教师的评论数据，进行教师教学评价自动分类。

项目描述：

1、构建文本分类测试集2万条，单条数据平均长度为100；

2、基于Bert系列预训练模型完成教学效果分类任务，共支持5种类别；

3、基于Bert-base-chinese和TextCNN模型进行知识蒸馏，badcase分析，模型所占空间缩小17倍，RT提升13倍，ACC指标89.3%；

项目职责：

1、基于业务需求，负责文本分类模型设计；

2、负责数据处理：制定标注规范、样本类别均衡度、badcase修复等；

3、负责模型搭建：基于RandomForest，FastText搭建baseline模型进行初步判断，在线任务模型Bert进行对比实验；

4、负责模型优化：知识蒸馏优化、badcase分析优化

5、负责分类模型API的开发，提供接口服务；



## 基于BERT+TextCNN模型的用户售后问题分类

项目描述：通过用户输入的问题描述进行售后问题分类，例如商品质量问题、物流问题、退换货问题等

所做事项：

1.对数据进行清洗，以及重构以符合fasttext的输入标准。

2.项目baseline解决方案的实现，选用随机森林和fasttext对项目评估基线。

3.负责模型的设计及优化，开始选用BERT系列，但是发现模型过于庞大，模型预测速度不理想，期间对模型尝试了剪枝、量化、知识蒸馏等处理，最终在发现使用textCNN时，模型达到了速度和精度的双平衡

## 消费者旅行反馈分类

项目描述：对不同的客户反馈进行分类有助于旅游企业掌握消费者的真实体验，得到不同人群的反馈和用户需求。同时根据消费者的意见进行相应的改进。

1.用户感受跟踪系统的子任务：对反馈进行分类。

2.采用fastext进行建模前的初判。

3.基于bert构建分类模型，进行多分类，acc指标93.6%。4.对bert模型进行量化压缩。

5.采用知识蒸馏提升模型预测效果。

工作描述:

1.对数据进行清洗和修正等处理。

2.利用不同的策略对模型进行优化。

3.采用知识蒸馏后相比于原来的bert模型大小缩小16倍。
