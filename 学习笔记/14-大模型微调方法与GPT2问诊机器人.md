## 一、大模型微调方法总览

### 1.1 NLP任务四种范式

| 范式 | 方法 | 特点 |
|------|------|------|
| 第一范式 | TF-IDF + 朴素贝叶斯等传统ML | 特征工程繁琐 |
| 第二范式 | word2vec + LSTM等深度学习 | 准确率提高，特征工程减少 |
| 第三范式 | **预训练模型 + Fine-Tuning**（如BERT） | 准确度显著提高，小数据集即可训练好模型 |
| 第四范式 | **预训练模型 + Prompt + 预测** | 训练数据显著减少，趋向少监督/无监督 |

### 1.2 Fine-Tuning（全量微调）

**基本思想**：采用已在大量文本上训练好的预训练语言模型，在小规模特定任务文本上继续训练，更新所有参数。

**痛点**：
- **参数规模过大**：全量微调更新所有参数，显存占用大、训练慢，每个下游任务都要保存完整模型权重
- **多任务效率低**：每个任务一份大模型，难以高效适配多场景

### 1.3 Prompt-Tuning（提示微调）

**核心思想**：让下游任务去迁就预训练模型，将Fine-Tuning的下游任务目标转换为Pre-training的任务形式。

**以情感分类为例**：
- 句子：`[CLS] I like the Disney films very much. [SEP]`
- 构建模板：`It was [MASK].` 拼接到原文后
- 输入BERT → 复用MLM分类器 → 预测 `[MASK]` 的词
- 标签映射（Verbalizer）："great"→positive，"terrible"→negative

**面向超大模型的Prompt-Tuning三大方法**：

#### (1) In-Context Learning（上下文学习）
- **Zero-Shot**：直接给任务描述 + 测试数据
- **One-Shot**：插入1个示例做指导
- **Few-Shot**：插入多个示例（10-100条）

#### (2) Instruction-Tuning（指令微调）
- 为每个任务设计指令模板，通过SFT（监督微调）让模型学会按指令生成输出
- Prompt激发**补全能力**，Instruction激发**理解能力**

#### (3) Chain-of-Thought（思维链）
- 在给出答案前，自动生成中间推理步骤
- Few-shot CoT：示例中包含推理过程
- Zero-shot CoT：直接用"Let's think step by step"激发推理

## 二、PEFT（参数高效微调）

**PEFT（Parameter-Efficient Fine-Tuning）**：仅微调少量或额外模型参数，固定大部分预训练参数，大大降低计算和存储成本。

### 2.1 Prefix-Tuning

**原理**：在模型输入前添加可训练的"虚拟tokens"序列（前缀），冻结PLM所有参数，只优化前缀参数。

**实现流程**：
1. 确定任务与基模型（GPT-2/BART/T5）
2. 设计prefix配置（`num_prefix`通常10-100）
3. 构造可训练参数矩阵 $P_\theta$，通过MLP重参数化
4. 在每层Transformer的注意力中，将prefix的key/value拼接到原始key/value
5. 冻结原模型参数，只对prefix做优化

**示例代码**（核心结构）：
```python
class SimplePrefixTuningEncoder(nn.Module):
    def __init__(self, prefix_len, num_layers, num_heads, head_dim):
        super().__init__()
        # 为每层维护可训练的 k 和 v 参数
        self.k_params = nn.Parameter(
            torch.randn(num_layers, prefix_len, num_heads * head_dim) * 0.02)
        self.v_params = nn.Parameter(
            torch.randn(num_layers, prefix_len, num_heads * head_dim) * 0.02)

    def forward(self, batch_size):
        outs = []
        for l in range(self.num_layers):
            k = self.k_params[l].unsqueeze(0).expand(batch_size, -1, -1)
            v = self.v_params[l].unsqueeze(0).expand(batch_size, -1, -1)
            # reshape 后拼接到原始 key/value
            outs.append((k.reshape(...), v.reshape(...)))
        return outs
```

**特点**：
- ✅ 只训练少量prefix参数，不同任务只需切换prefix
- ❌ 小模型表现差；占用输入序列长度；判别式任务常逊于LoRA

### 2.2 Adapter-Tuning

**原理**：在预训练模型内部的网络层之间插入adapter模块（bottleneck结构），只训练adapter参数。

**Adapter结构**：
```
输入 → LayerNorm → 降维Linear(down) → 激活函数 → 升维Linear(up) → 残差连接 → 输出
```

**示例代码**：
```python
class Adapter(nn.Module):
    def __init__(self, hidden_size, bottleneck_size):
        super().__init__()
        self.down_proj = nn.Linear(hidden_size, bottleneck_size)  # 降维
        self.act = nn.ReLU()
        self.up_proj = nn.Linear(bottleneck_size, hidden_size)    # 升维
        self.layer_norm = nn.LayerNorm(hidden_size)

    def forward(self, x):
        residual = x
        x = self.layer_norm(x)
        x = self.down_proj(x)   # 降维
        x = self.act(x)
        x = self.up_proj(x)     # 恢复维度
        return residual + x      # 残差连接
```

**冻结基模型，只训练Adapter**：
```python
def freeze_base_model_params(model):
    for p in model.parameters():
        p.requires_grad = False
    for module in model.modules():
        if isinstance(module, Adapter):
            for p in module.parameters():
                p.requires_grad = True
```

**特点**：
- ✅ 只训练少量adapter参数，多任务可共享大模型
- ❌ adapter容量有限；瓶颈层大小等超参数敏感；引入额外推理延迟

### 2.3 LoRA（Low-Rank Adaptation）

**核心思想**：通过低秩分解近似权重矩阵的参数更新。冻结预训练权重 $W_0$，用两个小矩阵 $A$ 和 $B$ 表示更新量：

$$\Delta W = B \times A \quad \text{其中} \quad A \in \mathbb{R}^{r \times d_{in}}, \ B \in \mathbb{R}^{d_{out} \times r}$$

前向传播变为：$y = W_0 x + \frac{\alpha}{r} \cdot B(Ax)$

**关键超参数**：
- `r`（秩）：通常4, 8, 16
- `alpha`（缩放系数）：常见 `alpha ≈ r` 或更大
- 通常加在Attention的Q/V投影矩阵上

**示例代码**：
```python
class LoRALinear(nn.Module):
    def __init__(self, orig_linear, r=8, alpha=16, dropout=0.1):
        super().__init__()
        self.orig = orig_linear  # 原始线性层（冻结）
        out, inp = orig_linear.weight.shape
        self.r = r
        self.scaling = alpha / r
        # A: (r, in), B: (out, r)
        self.A = nn.Parameter(torch.randn(r, inp) * 0.01)
        self.B = nn.Parameter(torch.zeros(out, r))  # B初始化为0
        self.dropout = nn.Dropout(dropout) if dropout > 0 else None

    def forward(self, x):
        base_out = self.orig(x)          # 原始权重输出
        down = x @ self.A.T              # 降维到r
        if self.dropout: down = self.dropout(down)
        up = down @ self.B.T             # 升维回out
        lora_out = self.scaling * up
        return base_out + lora_out       # 叠加增量
```

**训练后合并**（便于部署）：
```python
# W_merged = W + (alpha/r) * (B @ A)
W.data = W.data + (module.scaling * (module.B @ module.A))
```

**特点**：
- ✅ 只训练极少参数，效果接近全参数微调
- ✅ 不同任务LoRA模块可插拔
- ❌ 低秩分解对参数空间表达能力有限制

### 2.4 QLoRA（量化LoRA）

**核心创新**：在4-bit量化基础上结合LoRA，极大降低内存消耗。

**三大关键技术**：
1. **4-bit NormalFloat (NF4) 量化**：信息理论最优的量化方案，专为正态分布数据设计
2. **双量化 (Double Quantization)**：对量化常数再次量化，进一步减少内存
3. **分页优化器 (Paged Optimizers)**：GPU内存不足时将优化器状态分页到CPU RAM

**工作流程**：
- 基础模型权重 → 4-bit NF4量化（冻结）
- LoRA权重 A/B → 保持16-bit精度（可训练）
- 前向传播时4-bit权重即时反量化到16-bit
- 梯度只流向LoRA的A/B矩阵

**特点**：
- ✅ 单张RTX 3090/4090可微调65B~70B参数模型
- ✅ 性能与16-bit LoRA甚至全量微调相媲美
- ❌ 调试比标准LoRA更复杂

### 2.5 微调方法对比总结

| 方法 | 核心思路 | 训练参数 | 推理延迟 | 存储成本 |
|------|----------|----------|----------|----------|
| Fine-Tuning | 更新所有参数 | 全量 | 无额外 | 高 |
| Prefix-Tuning | 输入前加虚拟tokens | 极少 | 低 | 低 |
| Adapter-Tuning | 层间插入adapter | 少 | 中 | 低 |
| **LoRA** | 低秩分解权重更新 | 极少 | **无额外**（可合并） | **极低** |
| QLoRA | 4-bit量化+LoRA | 极少 | 无额外 | **极低** |

**预处理流程**（preprocess.py）：
1. 加载BertTokenizer分词器
2. 读取文本，用 `\n\n` 切分出每段对话
3. 对每段对话拼接问题和答案，编码为token id列表
4. 处理格式：`[CLS] 问题文本 [SEP] 答案文本 [SEP]`
5. 序列化保存为pkl文件

```python
def data_preprocess(txt_path, save_path):
    tokenizer = BertTokenizer.from_pretrained(vocab_path)
    sep_id = tokenizer.sep_token_id
    cls_id = tokenizer.cls_token_id

    with open(txt_path, 'r', encoding='utf-8') as f:
        data = f.read()
    data_list = data.split('\n\n')

    dialogue_list = []
    for dialogue in data_list:
        qa_list = dialogue.split('\n')
        input_ids = [cls_id]
        for sentence in qa_list:
            input_ids += tokenizer.encode(sentence, add_special_tokens=False)
            input_ids.append(sep_id)
        dialogue_list.append(input_ids)

    with open(save_path, 'wb') as f:
        pickle.dump(dialogue_list, f)
```

### 3.5 Dataset与DataLoader

**MedicalDataset类**：

```python
class MedicalDataset(Dataset):

        return len(self.data)

```

**collate_fn**（关键的批次处理函数）：
```python
def collate_fn(batch_data):
    """
    关键设计：
    - inputs用0填充（padding）
    - labels用-100填充 → GPT2会忽略-100位置，不参与loss计算
    """
    batch_data = [torch.tensor(data) for data in batch_data]
    inputs = pad_sequence(batch_data, batch_first=True, padding_value=0)
    labels = pad_sequence(batch_data, batch_first=True, padding_value=-100)
    return inputs, labels
```

> **重要概念**：`-100` 是PyTorch交叉熵损失的默认忽略索引（ignore_index），GPT2模型在计算loss时会自动跳过标签为-100的位置。

**单轮训练函数**：
```python
def train_epoch(data_loader, epoch, optimizer):
    total_loss = 0
    total_correct = 0
    total_num = 0

    for i, batch in enumerate(data_loader):
        inputs, labels = batch
        inputs = inputs.to(device)
        labels = labels.to(device)

        # 前向传播（传入labels自动计算loss）
        outputs = gpt(inputs, labels=labels)
        loss = outputs.loss

        # 反向传播
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()

        # 统计准确率
        batch_num, batch_correct = calc_correct(outputs.logits, labels)
        total_correct += batch_correct.item()
        total_num += batch_num.item()
        total_loss += loss.item()
```

**token准确率计算**：
```python
def calc_correct(logits, labels, ignore_index=-100):
    """
    logits: [batch_size, seq_len, vocab_size] 预测概率分布
    labels: [batch_size, seq_len] 标注token
    """
    preds = logits.argmax(dim=-1)         # 预测token
    not_pad = labels.ne(ignore_index)      # 非填充位置
    correct_position = preds.eq(labels)    # 预测正确位置
    correct = correct_position.masked_select(not_pad)  # 排除填充位
    return not_pad.sum(), correct.sum()
```

**训练主流程**：
```python
def do_train():
    train_loader, valid_loader = get_dataloader()
    optimizer = Adam(gpt.parameters(), lr=Config.lr)
    best_valid_loss = float('inf')

    for epoch in range(Config.epochs):
        train_epoch(train_loader, epoch, optimizer)
        eval_loss, eval_acc = evaluate_gpt2(valid_loader)
        print(f'epoch:{epoch}, eval_loss:{eval_loss}, eval_acc:{eval_acc}')

        # 保存验证loss最低的模型
        if eval_loss < best_valid_loss:
            best_valid_loss = eval_loss
            gpt.save_pretrained(Config.save_model_path)
```

**进阶训练技巧**（讲义完整版）：
- **梯度累积**：`gradient_accumulation_steps=4`，loss缩放后累积，等效大batch_size
- **梯度裁剪**：`clip_grad_norm_(model.parameters(), max_norm=2.0)` 防止梯度爆炸
- **学习率预热**：`get_linear_schedule_with_warmup`，先从小到大再逐渐衰减
- **Warmup步数**：`warmup_steps=100`

### 3.7 预测与推理

**解码策略**：
1. **Greedy Search（贪心解码）**：每步取概率最高的token
2. **Top-K Sampling**：从概率最高的K个token中采样
3. **Top-P Sampling（Nucleus）**：从累积概率达到P的最小token集合中采样

**推理参数**（讲义配置）：
- `max_history_len = 3`：历史对话长度
- `max_len = 300`：每个回答最大长度
- `repetition_penalty = 5.0`：重复惩罚
- `topk = 2`，`topp = 0.7`

## 四、核心知识点总结

1. **微调范式演进**：Fine-Tuning → Prompt-Tuning → PEFT（Prefix/Adapter/LoRA/QLoRA）
2. **LoRA是当前最通用的PEFT方法**：通过低秩矩阵A和B近似权重更新，训练后可合并回原模型，无推理延迟
3. **GPT2聊天机器人关键流程**：数据预处理（拼接QA为token序列）→ Dataset/DataLoader（-100忽略填充）→ 训练（传入labels自动计算loss）→ 评估（token级准确率）→ 推理（Top-K/Top-P解码）
4. **HuggingFace GPT2模型**：传入`labels`参数时自动计算loss，内部自动做logits/labels位移对齐，使用交叉熵损失且自动忽略-100
