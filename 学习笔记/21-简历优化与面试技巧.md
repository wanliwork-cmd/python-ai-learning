## 一、简历优化概述

### 1.1 简历的重要性

- 简历是求职的**第一印象**，决定面试机会的获取
- 突出个人能力和经验，提高求职成功率
- 更有效地展示自身优势，符合招聘者阅读习惯

### 1.2 优化原则

- **针对性**：根据目标岗位定制简历内容
- **简洁性**：突出核心信息，避免冗余
- **专业性**：用专业术语描述技术能力
- **真实性**：所有内容必须真实可信

## 二、简历结构与内容

### 2.1 个人信息与求职目标

- 姓名、联系方式、邮箱等基本信息
- 求职目标明确（如：NLP算法工程师、大模型开发工程师）

### 2.2 工作与教育经历

- 工作经历按**时间倒序**排列
- 突出与岗位相关的技术成果
- 教育背景重点展示学历和专业

### 2.3 项目经验

项目经验是简历的**核心部分**，需要重点描述：
- **业务场景**：项目解决什么问题
- **技术架构**：使用了哪些技术栈
- **个人职责**：自己负责了哪些部分
- **优化成果**：做了什么优化，效果如何

### 2.4 技能能力

- 编程语言（Python、SQL等）
- 框架工具（PyTorch、Transformers、Langchain等）
- 专业技能（NLP、大模型微调、RAG等）

### 2.5 自我评价与兴趣爱好

**自我评价撰写要点**：
- 与求职岗位的**匹配度**强调
- 凭借扎实的技术背景和对行业的深刻理解，能够迅速适应并积极贡献
- 拥有与所需岗位高度相关的项目经验，能有效推动项目进展
- 强烈的职业目标与对公司的热情

**兴趣爱好展示**：
- 选择与职业发展相关的兴趣爱好
- 如：阅读科技趋势书籍、参与开源社区等
- 安排在自我评价之后，使简历层次分明

## 三、面试流程

### 3.1 面试环节

```
自我介绍 → 项目介绍 → 技术提问
（2轮技术面 + 1轮经理面 + 1轮HR面）
```

### 3.2 面试基本要求

- **展示专业形象**：着装得体、仪态端庄、声音洪亮有底气
- **自我介绍**：简洁、有重点，突出关键技能、经验和成就
- **时间管理**：在规定时间内回答问题，展示良好的时间管理能力

---

## 四、回答问题框架

### 4.1 自我介绍框架

```
姓名 + 学历 + 毕业学校专业 + 工作经历 + 做了什么项目（简单介绍）
```

### 4.2 项目讲解框架

```
业务场景描述（解决什么问题）
    + 项目架构
    + 自己负责的部分
    + 优化点
```

### 4.3 知识点讲解框架

```
是什么 → 原理是什么 → 工作中怎么用 → 优化点
```

### 4.4 模型讲解框架

```
什么模型 + 什么数据 + 实现什么功能 
+ 模型特点（功能 + 架构 + 损失函数 + 输出）
+ 其他优化点
```

### 4.5 项目系统实施流程

```
总体实施方案 → 需求说明文档 → 项目设计文档 → 详细设计文档 
→ 代码实现 → 测试说明文档 → 上线 → 后期维护过程
```

## 五、反问技巧

### 5.1 反问技术面试官的问题

| 类别 | 问题示例 |
|------|----------|
| 技术栈 | 组内主要的技术栈/语言？ |
| 业务 | 部门具体业务/面试官的工作内容？ |
| 挑战 | 部门业务的挑战/难点？ |
| 岗位 | 如果有幸加入后可能负责的工作内容？ |
| 团队 | 部门的人数/人员构成？ |
| 规划 | 部门业务的后续规划？ |
| 培训 | 针对新员工有哪些培训？ |
| 体验 | 面试官的工作体验、工作强度？ |
| 流程 | 后续是什么流程？还有几轮面试？ |
| 反馈 | 您对我面试表现的评价？有哪些建议？ |

### 5.2 反问HR的问题

- 针对新员工有哪些培训？
- 对我有哪些评价或者建议？
- 大概多久能够出结果？
- 公司福利/三餐/房补

**注意**：HR面试时，不要直接问户口、薪资等问题，等HR主动谈起或决定录用后再沟通。

## 六、面试高频题

### 6.1 技术类高频题

| 题目 | 考查点 |
|------|--------|
| 项目背景、数据集来源 | 项目理解深度 |
| LoRA微调的原理及使用方式 | PEFT微调技术 |
| RAG的基本原理及优化方法 | 检索增强生成 |
| 项目中性能评估 | 评估指标理解 |
| 微调评估：BLEU、ROUGE、PPL | 文本生成评估 |
| RAG评估：RAGas（答案相关度、上下文召回率、忠实度） | RAG评估框架 |
| 其它评估：P、R、F1-score | 分类评估指标 |
| 基于Graph的RAG系统：GraphRAG、LightRAG | 前沿技术 |
| Transformer的基本原理 | 核心架构理解 |

### 6.2 非技术类高频题

- 离职原因
- 项目组情况
- 职业规划

## 十、面试准备建议

```
1. 简历准备
   ├── 根据目标岗位定制简历
   ├── 参考18份模板选择合适格式
   └── 重点突出项目经验和技术栈

2. 技术准备
   ├── 精读面试宝典（重点：Transformer、LoRA、RAG、大模型）
   ├── 复习课程项目（能讲清楚业务场景+架构+优化点）
   └── 刷算法题（两数之和、滑动窗口、动态规划）

3. 面试实战
   ├── 练习自我介绍（控制在2分钟内）
   ├── 用框架回答问题（是什么→原理→怎么用→优化点）
   └── 准备反问问题（技术面试官+HR各准备3-5个）

4. 信息收集
   ├── 牛客网查看目标公司面经
   ├── 脉脉了解部门评价
   └── InfoQ跟踪前沿技术动态
```
