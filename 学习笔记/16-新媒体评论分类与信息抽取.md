> **项目目标**：基于ChatGLM-6B大模型，通过LoRA微调实现新媒体评论的智能分类与SPO三元组信息抽取

## 一、项目概述

本项目是一个**多任务**系统，基于 ChatGLM-6B 大模型 + LoRA 微调，同时支持两类NLP任务：
- **文本分类**：对新媒体评论进行情感/类别分类
- **信息抽取（SPO）**：从文本中抽取主语-谓语-宾语三元组（Subject-Predicate-Object）

训练数据为混合数据集（`mixed_train_dataset.jsonl`，902条训练 + 122条验证），每条数据格式：
```json
{"context": "Instruction: ...\nInput: ...\nAnswer: ", "target": "输出结果"}
```

class ProjectConfig(object):

        self.use_lora = True
        self.lora_rank = 8           # 秩：8,16,32...
        self.lora_alpha = 32         # 缩放系数
        self.lora_dropout = 0.1

**关键要点**：
- `batch_size=2`：ChatGLM-6B 参数量大（约61.77亿），即使LoRA微调也需要控制batch_size防止OOM
- `lora_rank=8`：低秩矩阵维度，平衡表达能力与参数效率
- 同时支持LoRA和P-Tuning两种微调方式，通过配置切换

## 三、数据预处理（data_handle/data_preprocess.py）

### 3.1 核心函数 convert_example_chatglm

这是将原始数据转换为 ChatGLM-6B 模型输入格式的关键函数：

```python
def convert_example_chatglm(examples, tokenizer, max_source_seq_len, max_target_seq_len):
    tokenized_output = {'input_ids': [], 'labels': []}
    max_seq_length = max_source_seq_len + max_target_seq_len + 1
    
    for example in examples['text']:
        example = json.loads(example)
        context = example["context"]  # 输入文本（含Instruction和Input）
        target = example["target"]    # 期望输出
        
        # 1. 分别编码prompt和target
        prompts_ids = tokenizer.encode(text=context, add_special_tokens=False)
        target_ids = tokenizer.encode(text=target, add_special_tokens=False)
        
        # 2. 长度截断（source留1个[gMASK]位置，target留<sop>和<eop>位置）
        if len(prompts_ids) >= max_source_seq_len:
            prompts_ids = prompts_ids[:max_source_seq_len - 1]
        if len(target_ids) >= max_target_seq_len - 1:
            target_ids = target_ids[:max_target_seq_len - 2]
        
        # 3. 拼接：source_ids + [gMASK] + <sop/bos> + target_ids + <eop/eos>
        input_ids = tokenizer.build_inputs_with_special_tokens(prompts_ids, target_ids)
        
        # 4. 填充至最大长度
        pad_len = max_seq_length - len(input_ids)
        pad_input_ids = input_ids + [tokenizer.pad_token_id] * pad_len
        
        # 5. 构建labels（-100忽略索引）
        context_length = input_ids.index(tokenizer.bos_token_id)  # bos在target的第一位
        labels = [-100] * context_length + input_ids[context_length:]
        pad_labels = labels + [-100] * pad_len
        
        tokenized_output['input_ids'].append(pad_input_ids)
        tokenized_output['labels'].append(pad_labels)
```

### 3.2 ChatGLM-6B 输入格式详解

```
输入序列结构：
[prompt_tokens] [gMASK] [<sop/bos>] [target_tokens] [<eop/eos>] [pad...]

标签序列结构：
[-100, ..., -100] [bos, target_tokens, eop] [-100, ..., -100]
 ↑ 忽略prompt部分    ↑ 只对target部分计算loss    ↑ 忽略padding
```

**与GPT2问诊机器人项目的对比**：
- 两者都使用 `-100` 忽略索引，只对目标部分计算损失
- ChatGLM 使用 `[gMASK]` + `<sop>` 特殊token，GPT2使用 `[sep]`
- ChatGLM 的 tokenizer 有 `build_inputs_with_special_tokens()` 方法自动拼接

### 3.3 数据长度分析工具

```python
def get_max_length(tokenizer, dataset_file):
    """测试数据集最大的输入/输出tokens"""
    source_seq_len_list = []
    target_seq_len_list = []
    with open(dataset_file, 'r', encoding='utf-8') as f:
        for line in tqdm(f.readlines()):
            line = json.loads(line)
            source_len = tokenizer.encode(line['context'])
            target_len = tokenizer.encode(line['target'])
            source_seq_len_list.append(len(source_len))
            target_seq_len_list.append(len(target_len))
    # 输出 Max, Min, Avg, Middle 统计信息
```

用途：在设置 `max_source_seq_len` 和 `max_target_seq_len` 前，先分析数据分布，合理设定长度限制。

## 四、数据加载器（data_handle/data_loader.py）

```python
def get_data():
    # 1. 加载数据集
    dataset = load_dataset('text', data_files={
        'train': pc.train_path,   # 902条
        'dev': pc.dev_path        # 122条
    })
    
    # 2. 加载tokenizer
    tokenizer = AutoTokenizer.from_pretrained(pc.pre_model, trust_remote_code=True)
    
    # 3. 使用partial创建数据处理函数（固定tokenizer和长度参数）
    new_func = partial(convert_example_chatglm,
                       tokenizer=tokenizer,
                       max_source_seq_len=pc.max_source_seq_len,
                       max_target_seq_len=pc.max_target_seq_len)
    
    # 4. map()对每个batch执行数据处理
    dataset = dataset.map(new_func, batched=True)
    
    # 5. 创建DataLoader
    train_dataloader = DataLoader(dataset["train"], shuffle=True,
                                  collate_fn=default_data_collator,
                                  batch_size=pc.batch_size)
    dev_dataloader = DataLoader(dataset["dev"],
                                collate_fn=default_data_collator,
                                batch_size=pc.batch_size)
    return train_dataloader, dev_dataloader
```

**要点**：
- 使用 HuggingFace `datasets` 库加载 jsonl 文件
- `default_data_collator`：Transformers自带的默认数据整理器，自动将列表转为tensor
- `functools.partial`：优雅地固定函数参数，避免使用 lambda

### 5.2 显存优化技术

```python
    # 将模型转为float16精度，减少内存占用
    model = model.half()
    
    # 启用梯度检查点技术：用计算换显存，显著降低显存需求
    model.gradient_checkpointing_enable()
    
    # 确保输入层梯度正确传播
    model.enable_input_require_grads()
    
    # 禁用KV Cache，减少训练时内存占用
    model.config.use_cache = False
```

**三大显存优化技术**：
| 技术 | 原理 | 效果 |
|------|------|------|
| `model.half()` | float32 → float16 | 显存减半 |
| `gradient_checkpointing_enable()` | 前向传播时不保存中间结果，反向传播时重算 | 大幅降低显存，略增计算时间 |
| `use_cache = False` | 禁用KV Cache | 训练时不需要缓存历史 |

**CastOutputToFloat 工具类**：
```python
class CastOutputToFloat(nn.Sequential):
    """将模型输出从float16转回float32，确保损失计算精度"""
    def forward(self, x):
        return super().forward(x).to(torch.float32)
```

### 5.6 autocast混合精度技术

```python
from torch.cuda.amp import autocast

with autocast():
    # 自动选择合适的精度进行计算
    # 矩阵乘法用FP16（快、省显存）
    # 损失计算等关键步骤用FP32（保证精度）

```

## 六、模型保存（utils/common_utils.py）

```python
def save_model(model, cur_save_dir):
    if pc.use_lora:
        # 深拷贝模型，避免影响训练中的模型
        merged_model = copy.deepcopy(model)
        # 将LoRA参数合并到原始模型中
        merged_model = merged_model.merge_and_unload()
        merged_model.save_pretrained(cur_save_dir)
    else:
        model.save_pretrained(cur_save_dir)
```

**`merge_and_unload()`**：将LoRA的增量矩阵（A×B）合并回原始权重矩阵W，得到 W' = W + BA。合并后的模型推理时无需额外加载LoRA模块。

## 七、模型推理（inference.py）

```python
def inference(model, tokenizer, instuction, sentence):
    model.eval()
    with torch.no_grad():
        # 拼接输入格式
        input_text = f"Instruction: {instuction}\n"
        if sentence:
            input_text += f"Input: {sentence}\n"
        input_text += f"Answer: "
        
        # 编码并生成
        batch = tokenizer(input_text, return_tensors="pt")
        out = model.generate(
            input_ids=batch["input_ids"].to(device),
            max_new_tokens=max_new_tokens,
        )
        
        # 解码并提取答案
        out_text = tokenizer.decode(out[0])
        answer = out_text.split('Answer: ')[-1]
        return answer
```

## 八、关键技术总结

| 技术点 | 说明 |
|--------|------|
| ChatGLM-6B | 清华60亿参数大模型，支持中英双语 |
| LoRA微调 | 仅0.0594%参数参与训练（36.7万/61.77亿） |
| gradient_checkpointing | 用计算换显存，大幅降低内存需求 |
| autocast混合精度 | FP16+FP32混合，加速训练同时保证精度 |
| model.half() | float32→float16，显存减半 |
| -100忽略索引 | 只对target部分计算loss |
| merge_and_unload | 推理时将LoRA合并回原模型 |
| 多任务 | 同一模型同时支持文本分类和SPO抽取 |
