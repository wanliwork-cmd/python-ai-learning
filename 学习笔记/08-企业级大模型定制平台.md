## 一、大模型数据分析

### 1.1 财报分析

大模型可以直接理解和分析结构化数据（如表格、财报数据），通过 Prompt 引导模型进行数据洞察。

**示例：公司季度财报分析**

```
Super Mobile Inc. 季度财报数据:
| 指标 | Q3 2023 | Q3 2022 |
|------|---------|---------|
| 产品销售 | 67,184M | 70,958M |
| 服务销售 | 22,314M | 19,188M |
| 总净销售额 | 89,498M | 90,146M |
| 毛利润 | 40,427M | 38,095M |
| 净利润 | 22,956M | 20,721M |
```

**Prompt 设计思路**：将原始数据直接放入 Prompt，要求模型分析核心指标变化趋势、盈利能力、区域表现等。

### 1.2 图表数据分析

大模型可以进行表格数据的计算和分析：

```
社交媒体广告活动数据：
|广告系列|展示次数|点击次数|转化次数|总花费($)
|A|10,000|500|50|1,000
|B|15,000|750|55|1,500
|C|8,000|640|48|800
|D|20,000|1,200|60|2,400
```

**可分析的问题**：
- 各广告系列的点击率（CTR = 点击/展示）
- 各广告系列的转化率（转化/点击）
- 平均转化成本（总花费/转化次数）
- 投资回报率对比

### 1.3 数据分析 Prompt 设计要点

1. **提供完整数据**：以 Markdown 表格或 CSV 格式提供
2. **明确分析目标**：指定要分析的指标和维度
3. **要求结构化输出**：让模型按照指定格式输出分析结果
4. **引导推理过程**：要求模型展示计算步骤

## 二、多风格翻译机

### 2.1 项目概述

基于**讯飞星火大模型** + **Streamlit** 构建的多风格翻译 Web 应用，支持 5 种翻译风格：

- 默认风格
- 古文风格
- 学术风格
- 琼瑶风格
- 莎士比亚风格

### 2.2 技术架构

```
用户输入 → Streamlit Web界面 → 拼接风格Prompt → 星火大模型API → 翻译结果展示
```

- **前端**：Streamlit + streamlit_chat
- **后端**：SparkLLM_Thread（WebSocket 连接讯飞星火 API）
- **大模型**：讯飞星火 SparkLLM v3.1

### 2.3 核心代码

```python
import streamlit as st
from streamlit_chat import message
import SparkLLM_Thread

# 页面开场白
st.markdown("#### 您好, 我是多风格翻译官小星, 很荣幸为您服务。")

# 用户输入
user_input = st.text_input("请输入您需要翻译的英文文本:", key='input')

# 风格选择
but = st.radio("翻译风格:",
    ('默认风格', '古文风格', '学术风格', '琼瑶风格', '莎士比亚风格'),
    horizontal=True)

# 根据风格设定 Prompt 后缀
style_map = {
    '默认风格': '',
    '古文风格': ', 请按照古文风格进行翻译, 用古诗词的行文风格, 做到辞藻精炼, 可用典故。',
    '学术风格': ', 请按照学术风格进行翻译, 保持严谨认真的风格。',
    '琼瑶风格': ', 请按照琼瑶风格进行翻译, 意境优美, 充满诗情画意。',
    '莎士比亚风格': ', 请按照莎士比亚风格进行翻译，意境优美，带有浓厚的英伦腔。'
}
style = style_map.get(but, '')

# 构建 Prompt 并调用大模型
if user_input:
    text = user_input + "\n请将上述英文内容翻译为中文" + style

    # 调用星火大模型
    output = SparkLLM_Thread.main(
        uid='itcast',
        chat_id='itcast',
        appid='your_appid',
        api_key='your_api_key',
        api_secret='your_api_secret',
        gpt_url='wss://spark-api.xf-yun.com/v3.1/chat',
        question=[{"role": "user", "content": text}]
    )

    # 保存并展示结果
    st.session_state['past'].append(user_input)
    st.session_state['generated'].append(output)

# 展示对话历史
if st.session_state['generated']:
    for i in range(len(st.session_state['generated']) - 1, -1, -1):
        message(st.session_state["generated"][i], key=str(i))
        message(st.session_state['past'][i], is_user=True, key=str(i) + '_user')
```

### 2.4 讯飞星火 API 调用

**关键参数**：
- `appid`：应用 ID
- `api_key`：API 密钥
- `api_secret`：API 密钥对应的秘钥
- `gpt_url`：WebSocket 接口地址
- 获取地址：https://console.xfyun.cn/services/bm3

**通信方式**：WebSocket 长连接，支持流式返回

### 2.5 Streamlit 会话状态管理

```python
# 使用 st.session_state 管理对话历史
if 'generated' not in st.session_state:
    st.session_state['generated'] = []
if 'past' not in st.session_state:
    st.session_state['past'] = []
```

`st.session_state` 是 Streamlit 的会话状态，在页面刷新时保持数据不丢失。

## 三、Pygame 消消乐小游戏

### 3.1 项目概述

使用 Pygame 库开发一个 8×8 网格的消消乐游戏，作为大模型代码生成能力的展示。

### 3.2 核心实现

```python
import pygame
import random

# 游戏常量
GRID_SIZE = 8
COLORS = [RED, GREEN, BLUE, YELLOW, PURPLE, CYAN, ORANGE]

class Game:
    def __init__(self):
        self.grid = []
        self.score = 0
        self.generate_grid()

    def check_matches(self):
        """检查水平/垂直方向是否有3个及以上相同颜色的方块"""
        matches = []
        # 水平匹配检查
        for row in range(GRID_SIZE):
            for col in range(GRID_SIZE - 2):
                if self.grid[row][col] == self.grid[row][col+1] == self.grid[row][col+2]:
                    matches.extend([(row, col), (row, col+1), (row, col+2)])
        # 垂直匹配检查（类似）
        return list(set(matches))

    def remove_matches(self, matches):
        """移除匹配方块，每个+10分"""
        for row, col in matches:
            self.grid[row][col] = None
            self.score += 10

    def fall_blocks(self):
        """方块下落填补空缺，顶部生成新方块"""
        # 从底部向上处理每列
        for col in range(GRID_SIZE):
            # ... 下落逻辑 ...
            # 顶部补充新方块
            for row in range(GRID_SIZE):
                if self.grid[row][col] is None:
                    self.grid[row][col] = random.choice(COLORS)
```

### 3.3 游戏核心机制

1. **初始化**：随机生成 8×8 网格，确保初始无可消除组合
2. **选择交换**：点击两个相邻方块进行交换
3. **消除检测**：检查水平/垂直 3 个连续相同颜色
4. **下落补充**：消除后上方方块下落，顶部生成新方块
5. **连锁消除**：递归检查新产生的可消除组合

## 四、企业级大模型应用场景总结

### 4.1 主要应用方向

| 应用场景 | 技术方案 | 特点 |
|---------|---------|------|
| 数据分析 | Prompt + 结构化数据 | 财报分析、广告效果分析 |
| 翻译工具 | 大模型 API + Streamlit | 多风格翻译、实时交互 |
| 代码生成 | Prompt 描述需求 | 生成游戏、工具代码 |
| 文本分类 | Few-Shot Prompt | 金融文档分类 |
| 信息抽取 | Schema + JSON 格式 | 结构化数据提取 |
| 智能客服 | 角色设定 + 知识库 | 企业客服机器人 |

### 4.2 大模型平台对比

| 平台 | 通信方式 | 特点 |
|------|---------|------|
| Ollama（本地） | HTTP REST | 免费、隐私、需 GPU |
| 讯飞星火 | WebSocket | 中文优化、流式响应 |
| OpenAI API | HTTP REST | 效果最好、付费 |
| 百度文心 | HTTP REST | 中文理解强 |

### 4.3 Streamlit Web 开发要点

```python
# 基本组件
st.markdown()      # Markdown 文本
st.text_input()    # 文本输入框
st.radio()         # 单选按钮
st.session_state   # 会话状态管理

# 消息展示
from streamlit_chat import message
message(text, is_user=True/False)  # 聊天气泡
```
