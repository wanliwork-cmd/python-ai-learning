## 一、知识蒸馏概述

### 1.1 什么是知识蒸馏

知识蒸馏（Knowledge Distillation）是将**大模型（教师模型）**的能力迁移到**小模型（学生模型）**的技术。

```
大模型（教师）  ——高质量输出——→  训练数据  ——微调——→  小模型（学生）
```

**本项目蒸馏流程**：
1. **数据生成**：调用 DeepSeek-R1 API 对问题生成高质量回答（含思考过程）
2. **质量评估**：使用 Qwen2.5-72B 对生成的问答对进行质量评分
3. **模型训练**：使用 LLaMA-Factory 训练小模型
4. **高效部署**：使用 vLLM 部署训练好的小模型

### 1.2 为什么需要蒸馏

| 对比 | 大模型（如DeepSeek-R1） | 蒸馏后的小模型 |
|------|------------------------|--------------|
| 推理速度 | 慢 | 快 |
| 部署成本 | 高（需要大量GPU） | 低（单GPU即可） |
| 回答质量 | 高（有推理思考） | 较高（学习了教师风格） |
| API依赖 | 需要远程调用 | 本地部署 |

## 二、数据生成模块

### 2.1 调用DeepSeek-R1生成答案（getAnswerFromDeepseek.py）

#### 核心函数：callServer

```python
def callServer(prompt, model="deepseek-ai/DeepSeek-R1"):
    url = "https://api.siliconflow.cn/v1/chat/completions"
    headers = {
        "Authorization": f"Bearer ****",
        "Content-Type": "application/json"
    }
    payload = {
        "model": model,
        "messages": [{"role": "user", "content": prompt}],
        "temperature": 0.6,    # 温度：高→多样性好，低→稳定
        "max_tokens": 6000
    }
    
    response = requests.post(url, json=payload, headers=headers)
    
    # DeepSeek-R1的输出分为两部分：
    think = response.json()['choices'][0]['message']['reasoning_content']  # 思考过程
    result = response.json()['choices'][0]['message']['content']           # 最终结论
    
    return success, msg, think, result
```

**关键点**：DeepSeek-R1 的输出分为 `reasoning_content`（思考过程）和 `content`（最终答案）。我们需要**同时记录**两部分，让小模型学习 DeepSeek 的思考风格。

#### 批量处理函数：askBatch

```python
def askBatch(fileName, targetName):
    resultList = []
    dataList = loadRawDataSet(fileName)
    for data in dataList:
        success, msg, think, result = callServer(data["instruction"])
        if success:
            # 用<think>标签包裹思考部分
            data["output"] = "<think>\n" + think + "\n\n\n" + result
            resultList.append(data)
    writeData(resultList, targetName)
```

**`<think>` 标签的作用**：大模型善于学习格式性强的结构，用 `<think>` 标签将思考过程和最终答案分隔开，训练后的小模型也会学会这种输出格式。

## 三、生产者-消费者多线程模式（producerProcesserMode.py）

### 3.1 核心架构

API调用是典型的**IO密集型任务**，网络请求有2-5秒延迟，串行处理效率极低。

```
┌─────────────┐    ┌──────────┐    ┌─────────────┐
│   生产者     │ →  │  任务队列  │ →  │  消费者×N    │
│ (读取数据)   │    │ (Queue)  │    │ (调用API)    │
└─────────────┘    └──────────┘    └──────┬───────┘
                                          ↓
                                   ┌──────────┐    ┌──────────────┐
                                   │  结果队列  │ →  │  主线程写入    │
                                   │ (Queue)  │    │  (持久化存储)  │
                                   └──────────┘    └──────────────┘
```

### 3.2 生产者（含断点续传）

```python
taskQueue = Queue()    # 任务队列
resultQueue = Queue()  # 结果队列

def produceTask(fileName, targetName):
    # 断点续传：读取已完成结果，避免重复处理
    finishedSet = set()
    if os.path.exists(targetName):
        with open(targetName, "r", encoding="utf-8") as file:
            for line in file:
                data = json.loads(line.strip())
                if "id" in data:
                    finishedSet.add(data["id"])
    
    # 逐行读取（内存友好型）
    with open(fileName, "r", encoding="utf-8") as file:
        for line in file:
            data = json.loads(line.strip())
            if data['id'] not in finishedSet:
                while True:
                    if taskQueue.qsize() < 20:  # 队列长度控制
                        taskQueue.put(data)
                        break
                    else:
                        time.sleep(1)  # 队列满了就等
```

### 3.3 消费者（含重试机制）

### 3.4 多线程调度

```python
def askBatchMT(fileName, targetName, processorCount):
    # 1个生产者线程
    threading.Thread(target=produceTask, args=(fileName, targetName)).start()
    
    # N个消费者线程
    for i in range(processorCount):
        threading.Thread(target=consumTask, args=(i,)).start()
    
    # 主线程接收结果并实时写入
    with open(targetName, 'w', encoding='utf-8') as target:
        while True:
            data, success, msg, think, result = resultQueue.get(block=True)
            if success:
                data["output"] = "<think>\n" + think + "\n\n\n" + result
                target.write(json.dumps(data, ensure_ascii=False) + "\n")
                target.flush()  # 实时写入，防止数据丢失
```

**并发策略**：IO密集型用线程，计算密集型用进程。

```
    └── 输出格式: <think>思考过程
</think>

真实答案
    ↓
Step 3: 使用Qwen2.5-72B对问答对质量评分（0-9分）
    └── 过滤低质量样本
    ↓
Step 4: 使用LLaMA-Factory对小模型进行微调训练
    ↓
Step 5: 使用vLLM高效部署蒸馏后的小模型
    └── OpenAI兼容API格式对外服务
```

## 六、项目关键技术总结

| 技术点 | 说明 |
|--------|------|
| DeepSeek-R1 | 具有推理思考能力的大模型，输出分为thinking和content |
| SiliconFlow API | 提供DeepSeek-R1和Qwen2.5-72B的API服务 |
| `<think>` 标签 | 结构化输出格式，让小模型学会思考风格 |
| 生产者-消费者模式 | 多线程并发调用API，提高吞吐量 |
| 断点续传 | 崩溃重启时跳过已完成样本 |
| 重试机制 | API调用失败时自动重试（最多3次） |
| 质量评估 | 用更大模型（72B）评估蒸馏数据质量 |
| LLaMA-Factory | 训练框架，支持多种微调方式 |
| vLLM | 高效推理部署框架，PagedAttention技术 |
| Queue队列 | 线程间通信，控制内存使用 |
