## 一、项目背景

### 1.1 业务场景

金融领域数据大量激增，如何从繁杂数据中获取有效信息，帮助投资者或研究者减少决策失误，成为金融数据分析的热门话题。利用大模型可以实现：

1. **金融文本分类**：自动识别文档类型（新闻报道/财务报告/公司公告/分析师报告）
2. **金融信息抽取**：从文本中提取关键实体（日期、股票名称、开盘价、收盘价、成交量等）
3. **金融文本匹配**：判断两段金融文本语义是否相似

### 1.2 技术方案

- **核心方法**：Prompt Engineering + In-Context Learning
- **模型**：Ollama + Qwen 系列（qwen2:1.5b / qwen2.5:7b / qwen3:8b）
- **不需要训练**：通过精心设计的提示词引导模型完成任务

## 三、文本分类模块

### 3.1 核心实现流程

```
定义类别+示例 → 构建 Few-Shot 对话历史 → 拼接用户输入 → Ollama 推理 → 输出分类结果
```

### 3.2 关键代码

```python
import ollama

# 1. 定义类别和示例
class_examples = {
    '新闻报道': '今日，股市经历了一轮震荡...',
    '财务报告': '本公司年度财务报告显示...',
    '公司公告': '本公司高兴地宣布成功完成最新一轮并购交易...',
    '分析师报告': '最新的行业分析报告指出...'
}

# 2. 构建 In-Context Learning 对话历史
def init_prompts():
    class_list = list(class_examples.keys())
    pre_history = [
        {"role": "system", "content": f"你是一个文本分类器，分类到：{class_list}类别中。"}
    ]
    for _type, example in class_examples.items():
        pre_history.append({"role": "user", "content": f'"{example}"是 {class_list} 里的什么类别？'})
        pre_history.append({"role": "assistant", "content": _type})
    return {'class_list': class_list, 'pre_history': pre_history}

# 3. 推理
response = ollama.chat(
    model='qwen3:8b',
    messages=[*custom_settings['pre_history'],
              {"role": 'user', "content": f'"{sentence}"是 {class_list} 里的什么类别？'}]
)
```

## 六、项目经验总结

### 6.1 Prompt 工程核心模式

| 模式 | 适用任务 | 关键要素 |
|------|---------|---------|
| System 角色设定 | 所有任务 | 明确定义模型身份和行为 |
| Few-Shot 示例 | 分类/抽取/匹配 | 每个类别至少一个正确示例 |
| 格式约束 | 信息抽取 | 指定 JSON 输出 + 缺失处理 |
| 输出限制 | 文本匹配 | "只需要回答是/否" |

### 6.2 常见问题与解决

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 输出包含 markdown 代码块 | 模型默认行为 | 正则提取 ` ```json...``` ` |
| 分类不准确 | 示例不够典型 | 增加更多高质量示例 |
| JSON 解析失败 | 模型输出格式不稳定 | try-except 兜底 |
| 模型回答多余内容 | Prompt 约束不够 | 加强 System Prompt 限制 |

### 6.3 Ollama 调用最佳实践
