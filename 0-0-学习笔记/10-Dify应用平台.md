## 一、Dify 平台介绍

### 1.1 什么是 Dify

**Dify** 是一款开源的大语言模型（LLM）应用开发平台，融合了 **BaaS（Backend as a Service）** 理念，使开发者可以快速搭建生产级的生成式 AI 应用。

**核心特性**：
- 支持数百个模型的接入
- 直观的 Prompt 编排界面
- 高质量的 **RAG 引擎**
- 稳健的 **Agent 框架**
- 灵活的**流程编排**
- 提供易用的界面和 API

**BaaS 理念**：将常见的后端功能封装成即插即用的服务，开发者可以像搭积木一样快速构建应用后端。

### 1.2 Dify vs Coze

| 特性 | Dify | Coze |
|------|------|------|
| 开源 | 是（可本地部署） | 部分开源 |
| 数据隐私 | 完全本地化 | 依赖部署方式 |
| 部署方式 | Docker 自部署 | 云端/开源版 |
| 适用场景 | 企业级、需定制 | 快速搭建、原型验证 |
| 模型接入 | 支持更多开源模型 | 以字节系模型为主 |
| 工作流编排 | 支持 | 支持 |

## 二、Dify 安装部署

### 2.1 系统要求

- CPU >= 2 Core
- RAM >= 4 GiB
- Docker >= 24.0.0 + Docker Compose >= v2.20.0

### 2.2 Docker 基础知识

**Docker 核心概念**：
- **镜像（Image）**：应用的可执行包，包含运行所需的一切
- **容器（Container）**：镜像的运行实例
- **Docker Compose**：多容器编排工具

**常用命令**：
```bash
docker ps                  # 查看运行中的容器
docker images              # 查看本地镜像
docker-compose up -d       # 后台启动服务
docker-compose down        # 停止并删除服务
docker logs <container>    # 查看容器日志
```

### 2.3 Dify 安装步骤

```bash
# 1. 克隆 Dify 仓库
git clone https://github.com/langgenius/dify.git

# 2. 进入 docker 目录
cd dify/docker

# 3. 复制环境配置
cp .env.example .env

# 4. 启动服务
docker compose up -d

# 5. 访问 http://localhost/install 进行初始化
```

### 2.4 添加大模型

Dify 支持多种模型提供商：
- **OpenAI**：GPT-4、GPT-3.5 等
- **Ollama**：本地开源模型（Qwen、Llama 等）
- **Anthropic**：Claude 系列
- **国内模型**：通义千问、文心一言、智谱等

在 Dify 设置中配置模型 API Key 即可使用。

## 三、Dify 进阶功能

### 3.1 知识库与 RAG

**RAG（Retrieval-Augmented Generation）** 是 Dify 的核心能力之一，让大模型能够基于私有知识进行回答。

**知识库构建流程**：

```
上传文档 → 文本分段 → 向量化（Embedding） → 存入向量数据库
                                                    ↓
用户提问 → 向量检索 → 召回相关片段 → 拼接 Prompt → LLM 生成回答
```

**Dify 知识库功能**：
- 支持多种文档格式（PDF、Word、TXT、Markdown）
- 自动文本分段和索引
- 多种检索策略（向量检索、全文检索、混合检索）
- 可配置 Top-K 和相关性阈值

### 3.2 RAGFlow 集成

**RAGFlow** 是外部的高级 RAG 引擎，相比 Dify 内置的 RAG 有更好的文档解析能力。

**RAGFlow 优势**：
- 更强的 PDF 解析（表格、图片等）
- 更灵活的分段策略
- 更好的中文支持

**集成方式**：在 Dify 中通过 API 调用 RAGFlow 作为外部知识库。

### 3.3 工作流编排

Dify 支持可视化工作流编排，可以构建复杂的 AI 应用流程：

```
用户输入 → [条件判断] → [LLM节点] → [知识库检索] → [输出]
                ↓
         [代码执行] → [HTTP请求]
```

**常用节点类型**：
- **LLM 节点**：调用大模型
- **知识检索节点**：从知识库中检索信息
- **条件分支**：根据条件走不同路径
- **代码执行**：运行 Python 代码
- **HTTP 请求**：调用外部 API

## 四、项目案例：智能法律助手

### 4.1 项目概述

基于 Dify 构建一个智能法律助手，能够：
- 回答常见法律问题
- 基于法律条文提供专业建议
- 生成法律文书模板

### 4.2 实现步骤

1. **创建知识库**：上传法律法规文档
2. **设计 Prompt**：设定法律助手角色和回答规范
3. **编排工作流**：
   - 用户输入法律问题
   - 知识库检索相关法条
   - LLM 基于法条生成回答
   - 输出结构化建议
4. **测试与优化**

### 4.3 关键 Prompt 设计

```
你是一位专业的法律顾问助手。请基于以下法律条文回答用户的问题：

{检索到的法条}

要求：
1. 引用具体法条
2. 给出明确的法律建议
3. 如涉及具体案件，建议咨询专业律师
4. 使用通俗易懂的语言
```

## 五、Dify 核心概念总结

| 概念 | 说明 |
|------|------|
| 应用 | Dify 中的 AI 应用实例 |
| 知识库 | RAG 知识源，支持文档上传和向量检索 |
| 工作流 | 可视化编排的处理流程 |
| 模型供应商 | 接入的各种 LLM（OpenAI、Ollama 等） |
| 嵌入模型 | 用于文本向量化的模型 |
| 向量数据库 | 存储文档向量的数据库 |
| API | Dify 提供的 REST API，用于外部集成 |

### 5.1 Dify 应用类型

| 类型 | 说明 | 适用场景 |
|------|------|---------|
| 聊天助手 | 多轮对话式应用 | 客服、问答 |
| 文本生成 | 单次输入、生成文本 | 摘要、翻译、写作 |
| Agent | 自主决策+工具调用 | 复杂任务处理 |
| 工作流 | 多步骤编排流程 | 复杂业务流程 |
